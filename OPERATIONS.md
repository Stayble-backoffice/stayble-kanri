# OPERATIONS.md

「乗り換え民泊管理」の運用ルールをまとめます。  
OTA、PMS、宿泊者名簿、駆け付け、清掃・リネン、報告運用について、LPと実装の前提を整理します。

---

## 1. OTA運用

### 1.1 基本方針

既存OTAアカウントの引き継ぎは不可。  
Stayble側で新規にOTA運用環境を立ち上げる。  
オーナーは補助ホストになる。

LP表現:

```txt
OTA運用はStayble側で新しく整備。オーナー様には補助ホストとして必要な確認に参加いただきます。
```

FAQにも必ず入れる。

### 1.2 標準対応

- OTA初期立ち上げ
- 掲載文作成/修正
- Stayble標準撮影写真の反映
- Airbnbメッセージ対応
- Booking.comメッセージ対応
- レビュー返信
- 予約変更対応
- キャンセル対応
- 返金交渉
- 掲載内容修正
- 簡易的な価格調整
- カレンダー調整

### 1.3 カレンダー調整の意味

以下を含む。

- 予約受付可否の調整
- ブロック日設定
- 清掃不可日の反映
- メンテナンス日の販売停止
- 延泊/キャンセル後の清掃日調整
- OTA間の在庫ズレ防止
- ダブルブッキング防止
- 大型物件の同日入替制限

## 2. PMS

PMSは以下を想定。

- AirHost
- Beds24

用途:

- 予約管理
- 宿泊者名簿管理
- OTA/PMS情報の照合
- 宿泊記録の保存
- 清掃カレンダーとの連携

## 3. 宿泊者名簿・本人確認

### 3.1 Staybleがやること

ユーザー決定事項:

```txt
チェックイン前のパスポート画像提示
OTAの情報をパスポートと照合
OTA/PMS上の宿泊記録（帳簿）管理
```

通常運用では、警察対応を宿泊者名簿業務として前面に出さない。  
警察・消防・医療機関連絡は、あくまで緊急対応の文脈。

### 3.2 最低限の実務フロー

1. 予約後、宿泊者情報入力を依頼
2. 必要に応じてパスポート画像を回収
3. OTA/PMSの宿泊者情報と照合
4. 未提出者へ催促
5. チェックイン前に確認
6. 宿泊記録をPMS/OTA上で保管
7. 必要時に出力できる状態にする

### 3.3 注意

- 宿泊者名簿・本人確認は法令/ガイドラインの更新に注意
- LPでは詳細を出しすぎない
- 契約書/運用マニュアルで具体的に定める

## 4. 駆け付け対応

### 4.1 対応時間

通常現地対応:

```txt
8:00〜22:00
```

22:00以降:

```txt
法定管理上必要な最低限の緊急対応のみ
```

LP表現:

```txt
通常の現地対応は8:00〜22:00。22時以降は、火災・水漏れ・ガス・鍵トラブルなど緊急性の高い事案に限定して対応します。
```

### 4.2 法定緊急対応の対象

標準的に想定するもの:

- 火災
- 煙
- ガス
- 水漏れ
- 警察/消防/医療機関連絡
- 鍵トラブル
- 設備停止

### 4.3 軽微対応

以下も受ける。  
ただし、チェックイン前に知らせがあったか、滞在中の依頼かで料金が変わる。

- 備品追加
- タオル追加
- 家電操作説明
- 忘れ物回収/発送
- ゲスト都合の鍵紛失
- 家具移動
- 非緊急の現地確認
- 深夜の軽微依頼

### 4.4 料金

LPには具体金額を出さない。  
「固定料金制」とだけ表示。

契約前案内で出す内容:

- 固定料金
- 移動費実費
- 22時以降の深夜割増25%
- 作業1回1時間まで
- 超過料金なし

### 4.5 作業時間

```txt
1回1時間で打ち切り
超過料金なし
```

1時間で解決しないものは以下で対応。

- 状況確認
- 応急対応
- 関係機関連絡
- オーナー報告
- 必要に応じた専門業者手配案内

### 4.6 報告

解決/未解決に関わらず、オーナーへ後日報告する。

LP表現:

```txt
現地対応後は、解決状況にかかわらず対応内容をオーナー様へ報告します。
```

## 5. 清掃・リネン

### 5.1 清掃

清掃はStaybleセット契約。  
既存清掃会社の継続は不可。

LP表現:

```txt
清掃品質まで責任を持つため、清掃・リネンはStaybleセット契約です。
```

### 5.2 リネン

リネンは標準かつ必須。

方式:

- 都度集配型
- ストック型

物件の収納スペースや運用方針に応じて選択。

### 5.3 消耗品

消耗品サービスはオプション。

- 使う場合: 在庫管理・補充まで対応
- 使わない場合: 清掃時の在庫状況の報告のみ

---

## 6. 写真撮影

標準対応・無料。  
プロ撮影ではなくStayble標準撮影。

LPでは「写真撮影0円」と出してよいが、注釈を入れる。

```txt
※写真撮影はStayble標準撮影です。プロカメラマン撮影ではありません。
```

---

## 7. 報告

当面は以下で対応。

- OTA閲覧
- 報告書
- PMS/Stayble上の記録

清掃管理システム「Stayble」をオーナーへ必ず見せる必要はない。  
必要に応じて案内する。

---

## 8. 定期報告

住宅宿泊管理業者として必要な定期報告は行う。  
当面は既存ツール・PMS・報告書などで対応し、将来的にシステム化を検討する。

---

## 9. 退室要請・宿泊者対応権限

迷惑行為・重大な近隣苦情・危険行為などが発生した場合、必要な範囲でStaybleが判断・対応できるよう、管理受託契約で権限を持つ想定。

詳細は `CONTRACT.md` に整理する。
